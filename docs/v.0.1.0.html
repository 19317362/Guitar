インストール

Windows

Guitar-x.x.x-win32.zip を展開すると Guitar というフォルダが作成されます。これを任意の場所に移動します。

特にこだわりがなければ C:\Guitar で構いません。Windowsの流儀に従いたい方は、64ビット版Windowsなら C:\Program Files (x86) 、32ビット版Windowsなら C:\Program Files 、または、特定のユーザーだけで使用したい場合は C:\Users\username\AppData\Local に移動します。

Guitar.exe のショートカットを作成して、スタートメニューや、タスクバー、デスクトップ等の、好きな場所にショートカットを置きます。

Linux (Debian/Ubuntu/Raspbian)

Guitar-x.x.x-linux-xxx.tar.gz を展開すると Guitar というディレクトリが作成されます。展開する前に、あらかじめインストール先ディレクトリに移動しておくといいでしょう。推奨のインストール先は /opt です。

$ sudo mkdir /opt
$ cd /opt
$ sudo tar zxvf Guitar-x.x.x-linux-x86-xxbit.tar.gz

依存パッケージのインストール

$ sudo apt-get install git libxkbcommon-x11-0 libxcb-xkb1 libxcb-image0 libxcb-icccm4 libxcb-render-util0 libxcb-xinerama0

ライブラリパスの追加

ld.so.conf に /opt/Guitar を追加します。

$ sudo vi /etc/ld.so.conf

ただし、 ld.so.conf に以下のような記述がある場合（最近のDebian/Ubuntuではそうなっています）、

include /etc/ld.so.conf.d/*.conf

/etc/ld.so.conf.d というディレクトリの中にファイルを作って定義するのが推奨されます。その場合 /etc/ld.so.conf は編集せずに閉じます。

/etc/ld.so.conf.d/soramimi_jp_guitar.conf
/opt/Guitar

ファイル名は何でも構いませんが、拡張子は conf でなければなりません。これを /etc/ld.so.conf.d の中に入れておくと、自動的に読み込まれます。

定義を更新します。

$ sudo ldconfig

ランチャー（ショートカット）の作成

ホームディレクトリに .local/share/applications を作成しておき、そこに、ランチャー定義ファイルのリンクを作成します。

$ cd ~/.local/share/applications
$ ln -s /opt/Guitar/LinuxDesktop/Guitar.desktop

macOS

Guitar-x.x.x-macos.zip を展開すると Guitar (Guitar.app) が作成されます。ファインダーを開き、[アプリケーション]に移動します。
